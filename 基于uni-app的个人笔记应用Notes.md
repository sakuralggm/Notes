# 一. 项目背景
<a name="tdnGw"></a>
## **1. 社会背景**
<a name="uH7WV"></a>
### (1). 很多人做事没规划
即便宅在家里,，时间一大把， 也每天都浑浑噩噩，不知道干什么，或者手里事情一大堆, 但是不知道从何做起.，即便制定了一堆的计划，但是计划总是赶不上变化。<br />事情一拖再拖，什么都没有干，一天就过去了，躺在床上，带着愧疚感， 一边责备一边鼓励自己，第二天一定要“好好学习，好好工作“。<br />做事没有规划，就好像走路没有规划路线一样，容易多走回头路，多做无用功，会降低工作和学习的效率。所以不管做什么事都要先把目标计划列出来，然后规划好再实施。
<a name="Pejps"></a>
### (2). 很多人想随时记录感想但没有合适的载体
今天上午头脑突然冒出一个点子，本计划晚上作为日更内容，结果到了晚上绞尽脑汁却怎么也想不起来，很是后悔当时没有记录下来。<br />看来即使在我们眼里拥有天才大脑的诗人白居易、李贺，捕捉灵感都不是完全靠大脑的记忆，而是借助陶罐、锦囊这样的神器，何况今天的我们每天要面对海量碎片信息冲击，不可能靠一个普通大脑记下所有的灵感，还只有靠外界工具——一个方便的记事本小程序。
<a name="r7uiT"></a>
### (3). 很多人缺乏记录总结的能力
好记性不如烂笔头。每个人每天都要接收、阅读大量的信息，很难做到过目不忘、一劳永逸，如果不进行记录、摘抄、建档等及时处理，可能就像你所说的容易忘记。<br />建议在进行筛选、甄别、取舍基础上要对有价值信息或今后有应用潜质的信息内容进行有效梳理、归类、编目、建库、管理，便于日后的快递查找与利用。<br />关键在于学以致用。养成了勤阅读、勤记录、勤思考、勤总结的良好习惯后，关键还是要对所学内容和所积累经验知识的勤运用。
<a name="QL7sP"></a>
## **2. 设计初衷**
<a name="jSKV0"></a>
### (1). 快捷方便
开发出一款轻量级的记事本小程序，并尽量提高应用程序的功能性和舒适感，达到用户“随时、随地、随手”可用的效果。页面十分简洁，好懂！
<a name="yACyx"></a>

### (2). 功能强大但不繁琐
加粗、选择标题、字体颜色、字体背景色、上传图片……等工具都齐全，没有一些繁琐而不常用的功能，相较用word来说，更加方便快捷。
<a name="S5Zru"></a>
### (3). 适合多种场景
不仅可以使你随时记录自己的生活以及工作的点点滴滴，同时还可以当做事项清单、课堂笔记。

开发者虽极尽所能地实现了部分的交互动画，但受限于自身水平和时间，所设想的包括但不限于“长按动画”、“点击图标动画”没有实现。用户体验感方面仍有较大提高空间。
<a name="ql8Pi"></a>
### (2). 功能尚可扩展
功能仅仅局限于做笔记这一块。人们做笔记大部分是为了学习目的，可以再增加自律奖励、番茄时钟、DDL设置与提醒、每月笔记和学习时间统计和反馈等众多功能，做成一个集成大部分学习所需功能的学习助手。
<a name="IyT7C"></a>
### (3). 暂时不支持多用户合作
没有实现用户体系，目前笔记只能一个人使用，后续引入用户体系后可以实现推广使用。
<a name="Riex2"></a>
## **3. 心得体会**
<a name="Tm4Be"></a>
### (1). 基础不牢，地动山摇
首先，前端无论如何发展，终究绕不过HTML、CSS和JavaScript这三个，三剑客看起来比较简单，但是基础概念比较多，需要反复琢磨应用才行。只有自己的基础扎实了，后面去学新的框架、新的技术才能事半功倍。
<a name="UhrBK"></a>
### (2).实践出真知
其次，要多做练习，多写代码，从错误中学习，每个出色的web前端工作者的背后都是长时间的不懈努力。web前端初学者一定要调整好自己的心态，通过一行行代码的经验累积去提升。对于一个前端初学者而言，需要在当前的时间里大量的去练习去实践，从而形成一个良好的学习习惯。
<a name="chqhD"></a>
### (3). 团结就是力量
最后，团结就是力量，团结可以使团队力量更加强大，解决很多个人不能解决的问题。
